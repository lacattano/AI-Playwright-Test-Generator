# Issue Tracker

**Type:** Local markdown

**Location:** `BACKLOG.md` at the repo root.

Issues are tracked as markdown sections in `BACKLOG.md`. The file uses section headers to organize items:

- `## 🟡 Active Improvements` — work in progress or ready to start
- `## 🔴 Open Bugs` — confirmed bugs awaiting fix
- `## 🌟 Future Enhancements` — ideas needing design before implementation
- `## ✅ Shipped` — completed items (archive)
- `## ✅ Closed Bugs` — resolved bugs (archive)

New issues created by skills should be added under the appropriate section with an ID (e.g. `AI-XXX` for features, `B-XXX` for bugs).

**External PRs as triage surface:** No — this is a solo project.
