# Triage Labels

Mapping from canonical triage roles to strings used in `BACKLOG.md`:

| Role | String | Meaning |
|------|--------|---------|
| `needs-triage` | `🆕 new` | Maintainer needs to evaluate |
| `needs-info` | `❓ needs-info` | Waiting on reporter / more details |
| `ready-for-agent` | `🟡 ready-for-agent` | Fully specified, agent can pick up |
| `ready-for-human` | `👤 ready-for-human` | Needs human implementation |
| `wontfix` | `superseded` | Will not be actioned |

These are applied as status tags within issue entries in `BACKLOG.md`, not as GitHub labels.
